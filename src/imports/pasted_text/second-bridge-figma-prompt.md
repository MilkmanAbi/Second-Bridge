**Figma Prompt – Second Bridge UI/UX Design (MILK Framework)**

Design a **clean, modern, and humane web application interface** for **Second Bridge**, an open-source platform that connects users to community support services, lightweight organizational tools, and discussion forums. Use the **MILK principles** (Minimal, Inclusive, Lucid, Kind) as the guiding philosophy. The design should feel **calm, professional, and useful**, not overly cheerful, patronizing, or alarmist. Focus on **clarity, accessibility, and realistic workflows**.

---

### **1. General Style & Aesthetic**

* **Color Palette:** Neutral, soft, professional tones (avoid bright reds, neon, or alarmist colors). Use gentle accent colors to differentiate sections subtly.
* **Typography:** Clear, modern sans-serif fonts (legible at all sizes). Hierarchy: headings, subheadings, body text.
* **Spacing:** Generous whitespace to reduce cognitive load. Avoid clutter.
* **Iconography:** Minimal, meaningful icons. Avoid overly cute, illustrative, or emotional imagery.
* **Overall Feel:** Professional, calm, structured, inviting. Users should feel in control, not “watched” or infantilized.

---

### **2. Layout & Structure**

#### **Header**

* Site logo / project name (Second Bridge)
* Global navigation: Home, Services, Tracker, Community, About
* Optional search bar accessible globally

#### **Sidebar (persistent left or right)**

* Quick access to:

  * **Services Directory**
  * **Community / Forums**
  * **Personal Tracker / Notes**
* Display hotlines/resources discretely under Services, not in banners

#### **Main Content Area**

* **Home / Dashboard:**

  * Recent or relevant services (based on region or search)
  * Optional announcements, updates, or community threads

* **Service Detail Pages:**

  * Service name, description, contact info, tags
  * Options to add to tracker/bookmark
  * Share functionality controlled entirely by user

* **Tracker / Notes Interface:**

  * Minimal, Notion-style layout
  * Users can create private notes or share selectively
  * Search and filter notes by service, date, or tag

* **Community / Forums:**

  * Threads organized by region, topic, or issue
  * Users can post and reply anonymously
  * Thread view: clear post hierarchy, timestamps, minimalistic moderation cues

---

### **3. Interaction Principles**

* All interactions are **voluntary and transparent**.
* **Search & Filtering:** Keyword search with optional region/category filters; explain why results appear.
* **Service Bookmarking:** Easy add/remove; tracker keeps optional private or shared state.
* **Community Posts:** Anonymous by default; clear moderation boundaries.
* **Notes:** Flexible, private-first; optional sharing.

**Animation / Transitions:** Smooth, subtle transitions; no jarring popups or alerts.

---

### **4. Accessibility**

* Keyboard navigable, screen reader friendly
* Sufficient color contrast
* Clearly labeled form fields and controls
* Avoid reliance on color alone to communicate meaning

---

### **5. Components / UI Elements**

* **Buttons:** Clear, consistent, with hover/focus states
* **Input Fields:** Minimal, labeled, inline validation if needed
* **Cards / Lists:** For services, community threads, and notes
* **Modals:** Rare; only for essential actions, dismissible
* **Search Filters:** Collapsible, explainable, minimal

---

### **6. Functional Flows to Mock**

1. **Service Discovery Flow:** Search → Filter → View Service → Add to Tracker / Bookmark / Share
2. **Personal Tracker Flow:** View Tracker → Add Note → Tag / Share → Filter / Search Notes
3. **Community Interaction Flow:** Browse Threads → Create Thread → Reply → Optional Anonymous Sharing
4. **Navigation & Global Search:** Accessible from all pages; consistent, predictable

---

### **7. Deliverables in Figma**

* High-fidelity screens for:

  * Landing / Dashboard
  * Services Directory
  * Service Detail
  * Tracker / Notes
  * Community / Forum Listing & Thread View
* Component library: Buttons, inputs, cards, modals, sidebars
* Annotated design system: Color codes, typography, spacing, component behavior

---

**Notes / Design Philosophy Reminder:**

* Do **not** over-cheer or simulate mental health support. Treat the user as a capable human.
* Emphasize **clarity, autonomy, and transparency**.
* All recommendations and listings are **informational only**.
* Stick to MILK: Minimal, Inclusive, Lucid, Kind.
